## АБСОЛЮТНЫЕ ЗАПРЕТЫ (никогда не нарушать)

1. НЕ вставлять мета-комментарии в тело документа:
   - "[!]", "(!)", "[Требуется...]", "[уточнить]"
   - "TODO", "FIXME", "XXX", "HACK"
   - "**Примечание:**", "<!-- комментарий -->"
   - "Значение уточняется", "по требованию заказчика"

2. НЕ использовать неизмеримые характеристики:
   - НЕЛЬЗЯ: "современный", "высокопроизводительный", "надёжный"
   - НЕЛЬЗЯ: "по требованиям заказчика", "на усмотрение поставщика"
   - НУЖНО: "не менее 16 ГБ", "не более 65 Вт", "не ниже класса «А»"

3. НЕ упоминать бренды без "или эквивалент":
   - НЕЛЬЗЯ: "процессор Intel Core i7"
   - НУЖНО: "процессор Intel Core i7 или эквивалент"
   - Исключения — не бренды, это стандарты и интерфейсы:
     USB, HDMI, DDR4, DDR5, NVMe, Wi-Fi, Bluetooth,
     GOST, ISO, IEC, TCP/IP, PDF, DOCX, XLSX, ODF, XML,
     SATA, PCIe, DisplayPort, RJ-45, x86-64, ARM,
     IPS, VA, TN, OLED, ГОСт, ТР ТС
   - Исключения — отечественные ОС (обязательны по реестру Минцифры,
     "или эквивалент" уже подразумевается формулировкой
     "из реестра российских программ"):
     Astra Linux, ALT Linux, РЕД ОС, Альт, Rosa Linux

4. НЕ указывать устаревшие нормативные акты:
   - НЕЛЬЗЯ: ПП №878, №616, №925 (отменены с 23.12.2024)
   - НУЖНО для ТОВАРОВ: ПП №1875 от 23.12.2024
   - НУЖНО для ТОВАРОВ: ПП №719 от 17.07.2015
   - НУЖНО для ПО: ПП №1236 от 16.11.2015

5. НЕ применять нормативы одной категории к другой:
   - ПП №1875, №719 — только для ТОВАРОВ и РАБОТ
   - ПП №1236 — только для ПРОГРАММНОГО ОБЕСПЕЧЕНИЯ
     (лицензии, подписки, SaaS, всё что входит в реестр Минцифры)
   - Для УСЛУГ: профстандарты, ГОСТ Р на услуги, СанПиН
     (ПП №1875, №719, №1236 к услугам не применяются)

## ОБЯЗАТЕЛЬНАЯ СТРУКТУРА ДОКУМЕНТА

### Для ТОВАРОВ:
1. Общие сведения (наименование, заказчик, НМЦК, срок)
2. Требования к предмету закупки
3. Требования к доставке и пуско-наладочным работам
4. Гарантия качества и сервисное обслуживание
5. Тара, упаковка и маркировка
6. Место, сроки и условия поставки
7. Перечень нормативных документов
8. Приложения (по одному на каждую позицию номенклатуры)

### Для УСЛУГ:
1. Общие сведения
2. Предмет и цель контракта
3. Требования к исполнителю (квалификация, допуски, опыт)
4. Требования к результатам (измеримые показатели качества, SLA)
5. Условия оказания услуг (место, режим, отчётность)
6. Требования к материалам (если применимо)
7. Перечень нормативных документов
8. Приложения (при необходимости)
НЕ включать разделы "Тара и упаковка", "Требования к товару"

### Для ПО (лицензии):
1. Общие сведения
2. Требования к программному обеспечению
   - Функциональные требования
   - Наличие в реестре Минцифры: обязательно
   - Тип лицензии, срок, количество рабочих мест
3. Техническая поддержка
4. Требования к поставке и активации
5. Перечень нормативных документов (ПП №1236)
6. Приложения

## ПРАВИЛА ТАБЛИЦЫ ХАРАКТЕРИСТИК

Каждая строка таблицы ОБЯЗАНА содержать:
- Наименование характеристики (точное, без сокращений)
- Единицу измерения (ГБ, Вт, мс, дюйм, МГц, шт. и т.д.;
  или "—" если неприменимо)
- Значение (всегда измеримое: "не менее X", "не более X",
  "от X до Y")

Порядок строк: от общего к частному.
1-я группа: идентификационные (форм-фактор, тип, назначение)
2-я группа: процессор и вычислительные ресурсы
3-я группа: память (ОЗУ, ПЗУ)
4-я группа: интерфейсы и коммуникации
5-я группа: дисплей (если применимо)
6-я группа: электропитание
7-я группа: безопасность и сертификация
8-я группа: ПО (предустановленное)
9-я группа: гарантия

## ЗАПРЕЩЁННЫЕ КОНСТРУКЦИИ (статья 33 44-ФЗ)

ЗАПРЕЩЕНО → ПРАВИЛЬНАЯ ЗАМЕНА:
"по требованиям Заказчика"      → конкретное числовое значение
"на усмотрение Исполнителя"     → конкретное числовое значение
"уточняется при поставке"       → конкретное числовое значение
"согласно технической документации" → конкретный ГОСТ или значение
"не хуже аналогов"              → конкретное сравнительное значение
"определяется Производителем"   → конкретное числовое значение
"в ассортименте"                → перечислить конкретные позиции
"современный"                   → конкретное поколение/год выпуска
"высокопроизводительный"        → конкретные числовые показатели
