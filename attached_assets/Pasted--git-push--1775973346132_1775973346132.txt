═══════════════════════════════════════════════════════

1.1. Найти и удалить:
  - print() и console.log() оставленные для отладки
  - Захардкоженные URL типа "http://localhost:8000"
  - Захардкоженные токены, пароли, API ключи в коде

  - НЕТ "*" (wildcard запрещён в продакшене)
  - Есть конкретные домены: vercel.app, ваш домен

1.4. Проверить что все новые endpoints:

ШАГ 4 — ПРОВЕРКА ENDPOINTS
═══════════════════════════════════════════════════════

Запустить локально и проверить каждый endpoint:

GET  /api/health           → {"status":"ok","db":"ok"}
POST /api/auth/register    → создаёт пользователя, plan=trial
POST /api/auth/login       → возвращает JWT токен
GET  /api/user/status      → план, лимиты, дни
GET  /api/pricing          → список тарифов
POST /api/generate         → генерирует ТЗ, сохраняет в history
POST /api/qa-check         → проверяет текст, возвращает score
POST /api/qa-autofix       → исправляет и возвращает fixed_text
GET  /api/generations      → история пользователя
GET  /api/generations/{id}/download → отдаёт файл

Проверить сценарии ошибок:
  - Запрос без токена → 401
  - Исчерпан триал → 402 с reason
  - Неверные данные → 422 с понятным сообщением
  - Несуществующий ID → 404
