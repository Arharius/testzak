Добавь в приложение автоматический QA-аудит сгенерированного ТЗ.

═══════════════════════════════════════════════════════
ЧТО НУЖНО СДЕЛАТЬ
═══════════════════════════════════════════════════════

После каждой генерации ТЗ — автоматически запускать проверку
и показывать результат пользователю ДО скачивания документа.

═══════════════════════════════════════════════════════
БЭКЕНД — POST /api/qa-check
═══════════════════════════════════════════════════════

Принимает: {text: str}  — текст готового ТЗ
Возвращает: {
  score: int,           // 0-100
  passed: bool,         // true если score >= 80
  issues: [
    {
      level: "error"|"warning",
      code: str,
      message: str,
      suggestion: str
    }
  ]
}

ПРАВИЛА ПРОВЕРКИ (реализовать через regex + логику, без LLM):

--- ОШИБКИ (error) ---

1. code="brand_name"
   Найти бренды без "или эквивалент":
   Паттерны: Intel, AMD, Nvidia, Realtek, Samsung, Seagate,
   WD, Kingston, Corsair, Dell, HP, Lenovo, Cisco, Mikrotik,
   а также модели вида: [A-Z]{2,}[0-9]{3,}, I[3579]-\d{4,}
   Если найдено — error
   Suggestion: "Замените на функциональную характеристику или добавьте «или эквивалент»"

2. code="exact_value"
   Найти точечные числовые значения в характеристиках
   где НЕТ ≥ / ≤ / не менее / не более / от...до
   Паттерн: цифра + (ГБ|МГц|Вт|мм|кг|дюйм) без модификатора
   Suggestion: "Замените на «не менее X» или «от X до Y»"

3. code="emoji"
   Найти любые emoji в тексте (unicode ranges)
   Suggestion: "Удалите emoji — документ официальный"

4. code="meta_comment"
   Найти фразы: "требует уточнения", "необходимо проверить",
   "уточнить у заказчика", "требуется проверка", "TBD",
   "[заполнить]", "[указать]"
   Suggestion: "Замените заглушку конкретным требованием"

--- ПРЕДУПРЕЖДЕНИЯ (warning) ---

5. code="subjective_color"
   Найти цвет с оттенком: тёмно-серый, светло-бежевый,
   тёмно-синий и аналоги
   Suggestion: "Упростите до основного цвета или укажите RAL-код"

6. code="no_ktru"
   Если тип закупки = товар и нет упоминания КТРУ/реестра
   Suggestion: "Проверьте наличие позиции в КТРУ на ктру.рф"

7. code="no_warranty"
   Нет раздела о гарантии или слов: гарантия, гарантийный срок
   Suggestion: "Добавьте требование к гарантийному сроку"

8. code="no_country"
   Нет упоминания страны происхождения (для товаров)
   Suggestion: "Укажите требования к стране происхождения (ПП №1875)"

ПОДСЧЁТ SCORE:
- Начало: 100 баллов
- Каждая error: -15
- Каждая warning: -5
- Минимум: 0

═══════════════════════════════════════════════════════
ФРОНТЕНД — блок QA-результата
═══════════════════════════════════════════════════════

Показывать МЕЖДУ кнопкой "Сгенерировать" и кнопкой "Скачать".

Если score >= 80 (passed=true):
┌─────────────────────────────────────────┐
│ ✓ Документ соответствует требованиям    │
│ Оценка: 92/100                          │
│ [Скачать DOCX]  [Скачать PDF]           │
└─────────────────────────────────────────┘
Цвет рамки: зелёный

Если score < 80 (passed=false):
┌─────────────────────────────────────────┐
│ ⚠ Найдено замечаний: 3                  │
│ Оценка: 55/100                          │
│                                         │
│ ✗ Обнаружен бренд "Intel I219V"         │
│   → Замените на функциональную хар-ку   │
│                                         │
│ ✗ Точечное значение "512 ГБ"            │
│   → Замените на «не менее 512 ГБ»       │
│                                         │
│ △ Не указана страна происхождения       │
│   → Добавьте требование (ПП №1875)      │
│                                         │
│ [Исправить автоматически]  [Скачать всё равно]
└─────────────────────────────────────────┘
Цвет рамки: красный для error, жёлтый для warning

Кнопка "Исправить автоматически":
→ POST /api/qa-autofix {text}
→ Применить исправления и перегенерировать блок QA

═══════════════════════════════════════════════════════
БЭКЕНД — POST /api/qa-autofix
═══════════════════════════════════════════════════════

Принимает: {text: str}
Делает автоматические замены:

1. emoji → удалить
2. "512 ГБ" (без модификатора) → "не менее 512 ГБ"
3. "тёмно-серый" → "серый"
4. "[заполнить]", "[указать]" → удалить с предупреждением

НЕ трогать автоматически:
- бренды (нельзя угадать замену без контекста)
- мета-комментарии (нужен ручной ввод)

Возвращает: {
  fixed_text: str,
  auto_fixed: [str],   // список что исправлено
  manual_required: [str]  // список что нужно исправить вручную
}

ИНТЕГРАЦИЯ В СУЩЕСТВУЮЩИЙ ФЛОУ
═══════════════════════════════════════════════════════

В существующем endpoint /api/generate (или аналогичном):
1. Сгенерировать текст ТЗ как обычно
2. Автоматически вызвать qa_check(text)
3. Вернуть вместе с текстом:
   {text, docx_url, qa: {score, passed, issues}}

Кнопка "Скачать" активна всегда — пользователь сам решает.
QA — информирует, но не блокирует.
