# Развертывание «Генератор ТЗ» на бесплатном хостинге

Приложение упаковано в единый Docker-контейнер: FastAPI (backend) раздает собранный React (frontend) как статику. Один сервис, один порт.

---

## Вариант 1: Render.com (рекомендуется)

### Шаг 1. Подготовка репозитория
1. Загрузите проект на GitHub (публичный или приватный репозиторий).
2. Убедитесь, что в корне лежат файлы `Dockerfile` и `render.yaml`.

### Шаг 2. Создание сервиса
1. Зайдите на [render.com](https://render.com) и создайте аккаунт.
2. Нажмите **New +** → **Blueprint**.
3. Подключите GitHub-репозиторий с проектом.
4. Render автоматически найдет `render.yaml` и создаст:
   - веб-сервис `tz-generator` (Docker, бесплатный тариф)
   - базу данных PostgreSQL `tz-generator-db` (бесплатный тариф)

### Шаг 3. Настройка переменных окружения
В разделе **Environment** сервиса `tz-generator` задайте:

| Переменная | Значение |
|---|---|
| `DEEPSEEK_API_KEY` | Ваш ключ DeepSeek API |
| `SUPERADMIN_PASSWORD` | Пароль суперадмина |
| `DATABASE_URL` | Заполнится автоматически из Blueprint |

### Шаг 4. Деплой
Нажмите **Apply**. Render соберет Docker-образ и запустит приложение. Через 3-5 минут сервис будет доступен по адресу `https://tz-generator.onrender.com`.

## Вариант 2: Railway.app

### Шаг 1. Подготовка
1. Загрузите проект на GitHub.
2. Убедитесь, что в корне лежат `Dockerfile` и `railway.json`.

### Шаг 2. Создание проекта
1. Зайдите на [railway.app](https://railway.app) и войдите через GitHub.
2. Нажмите **New Project** → **Deploy from GitHub repo**.
3. Выберите репозиторий с проектом.
4. Railway автоматически обнаружит `Dockerfile` и начнет сборку.

### Шаг 3. Добавление базы данных
1. В проекте нажмите **New** → **Database** → **PostgreSQL**.
2. Railway автоматически добавит переменную `DATABASE_URL`.

### Шаг 4. Настройка переменных
В разделе **Variables** сервиса добавьте:

| Переменная | Значение |
|---|---|
| `DEEPSEEK_API_KEY` | Ваш ключ DeepSeek API |
| `SUPERADMIN_PASSWORD` | Пароль суперадмина |
| `PORT` | Оставьте пустым — Railway задает автоматически |

### Шаг 5. Деплой
Railway запустит сборку автоматически. Через 3-5 минут нажмите **Generate Domain** для получения публичного URL.

## Проверка работоспособности

После деплоя откройте в браузере:
- `https://<ваш-домен>/` — главная страница приложения
- `https://<ваш-домен>/health` — проверка здоровья (должен вернуть `{"status": "ok"}`)

---

## Архитектура деплоя

```
Docker-контейнер (1 сервис)
├── FastAPI (Python) — API на порту $PORT
├── static/ — собранный React (index.html, assets/)
└── PostgreSQL — внешняя БД (Render/Railway)
```

Фронтенд собирается на этапе Docker build (multi-stage: Node.js → Python). FastAPI раздает статику и обрабатывает API-запросы. Один контейнер, один порт, один сервис.

## Ограничения бесплатных тарифов

| Платформа | RAM | Диск | Спящий режим |
|---|---|---|---|
| Render Free | 512 МБ | — | Засыпает через 15 мин без запросов |
| Railway Free | 512 МБ | 1 ГБ | Лимит 500 часов/мес |

Для продакшена с постоянной доступностью рекомендуется платный тариф ($7/мес на Render, $5/мес на Railway).
