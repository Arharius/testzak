Исправь два критических бага в генерации ТЗ.

═══════════════════════════════════════════════════════
БАГ 1 — КОДИРОВКА DOCX (текст нечитаем в PDF)
═══════════════════════════════════════════════════════

В сгенерированном DOCX текст отображается кракозябрами
(вместо русских букв — латинские символы и цифры).

Причина: неверная кодировка при создании документа
через python-docx.

Исправить в функции создания DOCX:

1. Убедиться что Document() создаётся без параметров
   (не передавать encoding вручную):
   doc = Document()

2. Все строки перед вставкой — явно в unicode:
   text = str(text) если есть сомнения

3. Шрифт установить явно для каждого параграфа:
   from docx.shared import Pt
   from docx.oxml.ns import qn
   
   run = paragraph.add_run(text)
   run.font.name = 'Times New Roman'
   run.font.size = Pt(12)
   # Явно установить кириллический шрифт:
   run._r.rPr.rFonts.set(qn('w:cs'), 'Times New Roman')

4. При сохранении — не указывать encoding:
   doc.save(filepath)  # НЕ doc.save(filepath, encoding='...')

5. Проверить: открыть сохранённый файл в LibreOffice/Word
   → весь текст должен быть читаемым на русском

БАГ 2 — ПУСТЫЕ ХАРАКТЕРИСТИКИ ("по требованиям Заказчика")
═══════════════════════════════════════════════════════

Сейчас генерируется:
"Количество вычислительных ядер не менее 4;
архитектура и частотные параметры по требованиям Заказчика"

Это нарушение 44-ФЗ ст.33 — характеристики должны
быть измеримыми. "По требованиям Заказчика" запрещено.

ИСПРАВИТЬ В ПРОМТЕ К LLM:

Добавить в системный промт генерации характеристик:

"""
ЗАПРЕЩЁННЫЕ ФОРМУЛИРОВКИ — никогда не использовать:
- "по требованиям Заказчика"
- "по усмотрению Заказчика"  
- "на усмотрение поставщика"
- "согласно технической документации"
- "уточняется при поставке"
- "определяется производителем"
- "не хуже аналогов"
- любые формулировки без конкретного числового значения

КАЖДАЯ характеристика ОБЯЗАНА содержать:
- конкретное числовое значение с единицей измерения
- модификатор: не менее / не более / от X до Y

Для процессора ОБЯЗАТЕЛЬНО указать:
- Количество ядер: не менее X
- Тактовая частота: не менее X ГГц
- Объём кэша L3: не менее X МБ
- Архитектура: x86-64 (не "по требованиям")

Для оперативной памяти:
- Объём: не менее X ГБ
- Тип: DDR4 или DDR5
- Частота: не менее X МГц

Для накопителя:
- Объём: не менее X ГБ
- Тип: SSD (SATA/NVMe — указать)
- Скорость чтения: не менее X МБ/с

Если конкретное значение неизвестно — взять
МИНИМАЛЬНЫЙ разумный отраслевой стандарт,
но НЕ оставлять поле без числа.
"""

ПОСТОБРАБОТКА — после генерации проверить regex:

Найти паттерны и заменить:
r"(?:по требованиям|по усмотрению|на усмотрение)\s+\w+" 
→ вызвать повторную генерацию только этой характеристики

r"архитектура.*?(?:;|$)" без числа
→ заменить на "Архитектура: x86-64"

r"частот\w+.*?по\s+треб\w+"
→ заменить на "Тактовая частота: не менее 2.4 ГГц"

ПРОВЕРКА ПОСЛЕ ИСПРАВЛЕНИЯ
═══════════════════════════════════════════════════════

Сгенерировать тестовое ТЗ для "Системный блок, 1 шт."

В результирующем DOCX проверить:
□ Текст читаем на русском (нет кракозябр)
□ Нет фраз "по требованиям Заказчика"
□ У каждой характеристики есть число + единица
□ У каждой характеристики есть "не менее" или диапазон
□ Частота процессора указана в ГГц конкретным числом
□ Архитектура = "x86-64" а не описание
